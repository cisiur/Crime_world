# UI and Controls

## Design goals

The interface should support grand strategy play, city reading, planning, and long-term organization management.

It should be readable at a glance and optimized for map-based decision making.

## Main screens

- world map
- city map
- district overview
- organization panel
- character panel
- business panel
- operations panel
- police and pressure panel
- diplomacy and relations panel
- succession and family panel

## Map interaction

The map should support:
- zooming
- panning
- selecting buildings
- selecting districts
- selecting routes
- selecting targets for operations
- toggling overlays

## Useful overlays

- crime heatmap
- police presence
- corruption map
- business control
- influence map
- logistics map
- strategic value map
- territory ownership

## Controls

The player should be able to:
- pause time
- speed time up or slow it down
- assign operations
- manage personnel
- inspect organizations
- inspect businesses
- inspect pressure sources
- switch cities or regions

## UI principle

The player should not need to micromanage every criminal act.
The UI should make strategic planning easy and execution readable.
