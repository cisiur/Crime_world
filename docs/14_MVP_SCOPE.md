# MVP Scope

## MVP goal

The MVP must prove one thing:

Can the core loop of planning, expansion, pressure, and survival be fun enough for repeated play?

## MVP definition

The first playable version should be small, focused, and testable.

### Must have

- one playable city
- one player character
- one organization
- a handful of NPC leaders and key actors
- a simple real-world-inspired city map or placeholder city map
- a limited set of buildings
- a limited set of operations
- a limited set of businesses
- police pressure
- rival criminal groups
- territory or district control
- basic economy and income flow
- recruitment
- planning and execution of operations
- pause and time speed controls
- save and load

### Nice to have later

- succession
- imprisonment as a deeper meta-system
- multiple cities
- regional influence
- more detailed politics
- more detailed media effects
- more detailed corruption networks
- family simulation
- heirs and dynasties
- international expansion

### Not in MVP

- global map
- multi-country gameplay
- world politics
- deep family simulation
- elaborate dynasty gameplay
- full OSM-based city generation for every city in the world
- hundreds of business types
- dozens of different organization archetypes
- advanced intelligence services
- complex banking systems

## MVP recommendation

The best MVP approach is:
1. build the core loop in one city,
2. use a manageable map and simulation scope,
3. prove that the player wants to plan one more operation and one more expansion,
4. only then scale to real cities through OpenStreetMap generation.

## Important principle

The OSM feature is a major selling point, but it should not be the first technical dependency of the project.
The game must first be fun on a controlled map.
